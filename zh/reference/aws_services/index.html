使用 AWS 云服务

本节介绍 Service Workbench 使用的一些 AWS 云服务。资源名称通常包括Namespace，包括部署时使用的stage name。如果您对每个部署使用不同的阶段名称，则可以从同一帐户部署多个 Service Workbench 实例。

###亚马逊EC2

Amazon EC2 仅用作部署 Service Workbench 的平台。有关更多详细信息，请参阅 部署实例 部分。

AWS IAM 角色​

Service Workbench 在您的帐户中创建多个角色。运行 部署后 SDC 时会创建角色 <namespace>-prep-raas-master-MasterRole-XXX。此角色与您部署 Service Workbench 的主帐户具有信任关系。有两种策略允许主帐户在此主帐户中担任角色。 帐户结构 定义了每种类型的帐户。 图 32 显示 AWS IAM “信任关系”选项卡。

图 32：AWS IAM 信任关系选项卡

外部 ID 与该角色相关联。外部 ID 是创建角色后提供的标识字符串。为了让受信任实体（您的主账户）在主账户中承担其角色，它必须提供此外部 ID。提供的外部 ID 在受信实体和管理帐户之间建立可撤销的关系。

在当前的 Service Workbench 部署中，外部 ID 在以下字符串工作台中配置为默认值：

主/解决方案/prepare-master-acc/config/settings/.defaults.yml

要更改此值，请在同一目录中创建一个以阶段命名的配置文件 (mystagename.yml)。有关详细信息，请参阅 配置 部分。下图显示了定义 Trusted Entities 如何承担角色的条件的屏幕截图。

图：定义可信实体的条件

AWS 组织​

在 Master 账户中创建了一个 AWS 组织。管理帐户在 帐户结构 部分中进行了更详细的讨论。 AWS 组织使用管理账户为每个部署创建一个单独的账户。该帐户的名称是使用的舞台名称。下图显示了 AWS Organizations “Accounts”选项卡的屏幕截图。

图：AWS Organizations 账户页面

亚马逊 S3​

Service Workbench 创建了多个 Amazon S3 存储桶。按 Stage Name 过滤显示部署的 Amazon S3 存储桶。下图显示了用于 Service Workbench 部署的 Amazon S3 存储桶。

图：用于 Service Workbench 部署的 Amazon S3 存储桶

'studydata' 存储桶包含此部署中个人和组织级别的各种 Studies 的所有数据。下面显示了 studydata 存储桶中内容的图像。

图：Amazon S3 StudyData 存储桶

AWS 成本管理器​

Service Workbench 能够显示在管理帐户下运行的工作空间所产生的实际成本。这是在 AWS 管理控制台中使用 AWS Cost Explorer 服务。 AWS Cost Explorer 必须为每个管理账户手动设置一次，以允许处理成本数据请求。进行此设置需要在主账户中完成后台进程，这可能需要长达 24 小时。
