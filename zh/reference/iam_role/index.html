参考-添加一个IAM role 到 EC2 instance

AWS 上的 Service Workbench 与多个 AWS 资源交互，包括 Amazon EC2、AWS IAM、AWS Organizations 等。您可以轻松地将 AWS IAM 角色添加到 Amazon EC2 实例，利用我们的 AWS Organizations 和您的账户结构，并创建多个 Amazon S3 存储桶。

将 AWS IAM 角色添加到 Amazon EC2 实例​

可以为 Amazon EC2 实例分配一个包含 AWS IAM 角色的实例配置文件。 AWS IAM 角色为 Amazon EC2 实例提供一组权限。 Amazon EC2 实例仅执行其 AWS IAM 角色定义的操作。向 Amazon EC2 实例添加 AWS IAM 角色可让您的应用程序安全地进行 API 调用，从而无需管理安全凭证。

Service Workbench 部署应用程序必须能够创建 AWS 资源。满足此要求的最简单方法是为 Amazon EC2 实例提供管理员角色。

向新的 Amazon EC2 实例添加管理员角色​

在为 Service Workbench 部署创建新的 Amazon EC2 实例时，可以将实例配置文件分配给 Amazon EC2 实例。选择位于 AWS IAM 角色下拉列表旁边的创建新的 IAM 角色。下图显示了 AWS 管理控制台中的创建新 IAM 角色操作的图像。

图 25：创建新的 AWS IAM 角色

要继续该过程，请突出显示 Amazon EC2 并继续访问权限。

  1. 在“Permissions”中，过滤“AdministratorAccess”并选择它。

图：Amazon EC2 实例中的权限

图：过滤管理员访问权限

  1. 继续“标签”。
  2. 在“审核”页面，输入角色名称。

图：为 Amazon EC2 实例选择角色名称

  1. 在 Amazon EC2 选项卡中，刷新 AWS IAM 角色下拉菜单，然后选择要附加到新 Amazon EC2 实例的管理员角色。

图：选择 Amazon EC2 实例的管理员角色

  1. 创建一个 Amazon EC2 实例。

将角色添加到现有实例​

要将角色添加到已运行的 Amazon EC2 实例，请在 EC2 控制台中选择 Amazon EC2 实例。打开 Action > Instance Settings 菜单，然后选择 Attach/Replace IAM Role。下图显示了Instance Settings 菜单。

图：在 EC2 控制台中附加/替换 AWS IAM 角色

在 Attach/Replace IAM Role 屏幕中，搜索您创建的角色，选择它，然后选择 Apply。下图显示了您可以附加/替换 IAM 角色的屏幕。

图：AWS IAM 角色搜索
