准备 organizational account

注意：仅当 Service Workbench 用于通过使用创建账户来出售 AWS 组织中的账户时，才需要此部分。如果 Service Workbench 仅使用使用 Add Account 加入的计费帐户，则可以省略此部分。

在此步骤中，您在 main/solution/prepare-master-acc 目录中部署 prepare_master_acc 可单独部署的组件 (SDC)。这会在允许“AssumeRole”的组织帐户中创建一个角色，并将主帐户作为其受信任的实体。如果您已在组织账户中部署了 Service Workbench，则主账户也是组织账户，可信实体与组织账户的账户 ID 相同。

此步骤中的默认设置为：

  • 主账户 ID：当前 AWS 账户 ID
  • 外部 ID：字符串 workbench

如果您将 Service Workbench 部署到组织帐户，这些默认值就足够了，并且默认配置文件具有组织帐户的权限，因为主帐户也是组织帐户。如果将 Service Workbench 部署到与组织账户不同的主 AWS 账户，您必须创建一个配置文件来指定主账户 ID 和要使用的配置文件。此配置文件必须具有组织帐户的权限。

创建配置文件​

如果在当前默认配置文件访问之外的帐户中部署 Service Workbench，请通过将 example.yml 复制到<stage>.yml。根据需要编辑文件：

  • awsProfile：具有主账户权限的 AWS 凭证配置文件。
  • mainAccountID：部署解决方案的主要 AWS 账户的 12 位 AWS 账户 ID。
  • externalId：根据需要。字符串 workbench 经常被使用。在 Service Workbench 中创建 AWS 账户时将需要此字符串。

在组织账户中启用 AWS 组织​

在 AWS 管理控制台 中，导航到 AWS Organizations 以确保组织账户存在组织。如果没有，则创建一个新的。没有配置设置。 Service Workbench 在 AWS Organizations 中为此部署创建一个新账户，以部署时使用的阶段名称命名。

部署准备主账户 SDC​

要部署 prepare_master_acc SDC，请执行以下操作：

  1. 阅读文件：main/solution/prepare-master-acc/README.md。
  2. 从目录 main/solution/prepare-master-acc 部署 orhanizational 帐户 SDC：
    pnpx sls deploy --stage <stage>
  1. 显示同一目录中组织角色的 ARN：
    pnpx sls info --verbose --stage <stage>

在 Service Workbench 中添加账户时将需要组织角色 ARN。

注意：运行便利脚本 scripts/master-account-deploy.sh <stage> 将执行与上述 pnpx sls deploy 相同的步骤。
